# Review
- 제약 사항이 20 under면 모든 경우의 수를 다 시도해보면 되겠구나! -> 리커전
  - 10 이하면 다 돌려도 되겠구나!
  - 20 이하면 가지치기를 통해 다 돌려도 되겠구나!
- 괄호가 추가될 때마다 -> Integer ++
- 괄호를 닫을 때마다 -> Integer --
- 단, Integer 값은 0 이상이어야 닫을 수 있도록..

## 결론
### 리커전 문제(이모티콘, 괄호, 케이크 등등)는 친숙해지자 😊
### 시간 복잡도 : 2^n
## 풀이
- 로직 자체는 oc < n, sum > 0의 조건문이 없어도 돌아가지만, 로직이 빠르게 회전할 수 있도록 추가
